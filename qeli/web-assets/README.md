# qeli web-panel assets

The admin panel ships **self-hosted** CSS/JS/fonts — no runtime CDN — so it works
on an air-gapped server reached over an SSH tunnel. The generated stylesheet and
the vendored Alpine.js / woff2 fonts live under [`../src/web/assets/`](../src/web/assets/)
and are embedded into the binary via `include_*!` (served from `/assets/*`).

## Regenerate the CSS

Edit a template under `../src/web/templates/`, then rebuild the Tailwind output:

```sh
cd qeli/web-assets
npm install      # first time only
npm run build    # → ../src/web/assets/app.css (minified)
# npm run watch  # rebuild on save while developing
```

`tailwind.config.js` scans `../src/web/templates/**/*.html` for utility classes;
the component layer + design tokens + `@font-face` live in `input.css`.

CI does **not** need Node — `app.css` is committed. Only regeneration needs this
package (`node_modules/` is git-ignored).

## Vendored assets (not generated by Tailwind)

- `alpine.js` — Alpine.js 3.13.5 (`cdn.min.js`)
- `fonts/inter-*.woff2` — Inter (latin, weights 400–800), from `@fontsource/inter`
- `fonts/jbmono-*.woff2` — JetBrains Mono (latin, 400/600), from `@fontsource/jetbrains-mono`

Re-vendor by downloading the same files into `../src/web/assets/` (see git history
for the exact URLs).
