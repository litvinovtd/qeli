# qeli — 2026-06-12 audit and fixes (release 0.7.1)

A review of an external audit (CRITICAL/HIGH/MEDIUM/LOW). Most items turned out to be
false / overstated — they analyze functions in isolation, without the surrounding
context (the inner AEAD that calls `ReplayGuard`, transcript binding, the trusted
config). Four were genuinely confirmed and fixed: **H-1, H-5, H-3, M-13** — all shipped
in **0.7.1**.

Legend: ✅ implemented + validated.

## M-13 — replay window 64 → 2048 ✅
Receiver-only, no wire break. The 64-bit window was easily knocked out by reordered UDP
→ false `ReplayDetected`. Replaced with a 2048-bit window (WireGuard-sized), a 32-word
array with a multi-word shift.
- **Rust** `protocol/packet.rs` — `[u64; 32]` + `shift()` + 2 tests. **C#** `PacketCodec.cs`
  — `ulong[]` + `ShiftWindow`. **Kotlin** `PacketCodec.kt` — `LongArray` + `ushr`.

## H-5 — atomic resolv.conf write without symlink-follow ✅
`client/dns.rs::write_atomic` — a predictable name + the symlink-following `std::fs::write`
let a local attacker redirect the target. Fix: **random name + `O_EXCL` + `O_NOFOLLOW`** +
collision retry. Rust client.

## H-3 — sanitizing values in kill-switch nft rules ✅
`client/killswitch.rs` — `tun_if` is validated as a Linux ifname (`[A-Za-z0-9_-]{1,15}`),
`ips` are reformatted from a parsed `IpAddr`. Defense-in-depth. Rust client.

## H-1 — binding data keys to the server's static identity ✅ (Rust + C# + Kotlin)
**A valid crypto observation:** the 0.7.0 session keys derive from the ephemeral DHs
(`ee`, `mlkem`); the server static key is NOT folded in → a failed ephemeral RNG exposes
the tunnel (weaker than Noise IK). **Fix:** fold `es = X25519(client_eph, server_static)`
into the KDF — an attacker must break BOTH DHs AND obtain the private static key.

**Design:** a separate KDF variant with its own HKDF salt (`…-v1-static-bound` /
`…-v2-hybrid-static-bound`) — no silent bound↔unbound interop: a mismatch yields different
keys and an honest failure. `es` is symmetric (server: `static_priv × client_eph`; client:
`client_eph_priv × pinned_static`) → both sides derive the same keys.

**Implementation (byte-identical):**
- **Rust** `crypto/derive.rs` (`derive_keys_bound`/`derive_keys_hybrid_bound` + the e2e test
  `bound_handshake_keys_agree_end_to_end`), server `handler.rs`/`udp_handler.rs`, client
  `client/mod.rs` (`static_es` + all KDF sites), config `config/{server,client}.rs`.
- **C#** `qeli-shared/Crypto/KeyDerivation.cs` (bound variants), `Model/VpnConfig.cs`
  (flag + parsing), `Vpn/VpnTunnelBase.cs` (`StaticEs` + 4 sites).
- **Kotlin** `crypto/KeyDerivation.kt`, `model/Config.kt`, `QeliService.kt` (`staticEs` +
  `makeCodecs*` taking `es`). Gotcha: `clientKeyPair.privateKey` is a `java.security.PrivateKey`.

### ⚠️ Since 0.7.1 H-1 IS ON BY DEFAULT
The flag `auth.bind_static_to_session` (server) and `bind_static` (client, the `[qeli]`
section) **default to `true`**. Consequences:
- **WIRE-BREAKING**: a server with H-1 only admits H-1 clients with a pinned key. Any
  non-H-1 client (TOFU + the whole deployed 0.7.0 fleet) stops connecting until it is
  upgraded. **Upgrade the server and all clients in lockstep.**
- An **unpinned / TOFU** client (all-zero `key`) must set `bind_static = false` explicitly,
  otherwise the handshake fails with a clear error (you can't derive `es` without a real
  key). The `client.conf` and `release/keenetic/client.conf.example` templates carry
  `bind_static = false`.
- The default **secure servers** (`server-maxobf`, `server-reality`, `prod-*`) and pinned
  clients carry the flag explicitly. `qeli://` links don't carry it — the client enables
  H-1 from its own default (on), and the pin travels in the link.
- For a staged upgrade from a legacy 0.7.0 fleet, set `false` on both sides, then enable.

## False / negligible audit items (for reference)
- **C-1/C-2** (obfs) — XOR/ChaCha20 masking is an anti-DPI layer, not a security layer
  (it sits on top of a real AEAD tunnel); "weak obfuscation" is by design.
- **C-3** — `ReplayGuard` IS wired (`server/reality.rs:84`).
- **M-9** — DNS LB is correct. **M-14** — a window reset doesn't enable replay.
- **SHA-1** — that's the WebSocket accept per RFC 6455, not crypto. **JOIN_MAGIC** — inside
  the AEAD, not on the wire. **thread_rng** — only for a nonce seed; keys use `OsRng`.
- **M-4** (O(n) pool), **M-6** (IPv6 in split_host_port) — minor/by-design, optional.
