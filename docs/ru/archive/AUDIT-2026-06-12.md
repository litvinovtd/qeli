# qeli — аудит 2026-06-12 и фиксы (релиз 0.7.1)

Разбор внешнего аудита (CRITICAL/HIGH/MEDIUM/LOW). Бóльшая часть пунктов оказалась
ложной/завышенной — это анализ функций в изоляции, без окружающего контекста
(внутренний AEAD, вызывающий `ReplayGuard`, transcript-binding, доверенность конфига).
Реально подтверждены и внесены четыре: **H-1, H-5, H-3, M-13** — все вошли в **0.7.1**.

Легенда: ✅ внесено+провалидировано.

## M-13 — replay-окно 64 → 2048 ✅
Receiver-only, без слома провода. 64-битное окно легко выбивалось переупорядоченным
UDP → ложные `ReplayDetected`. Заменено на 2048-битное (WireGuard-размер), массив из
32 слов с многословным сдвигом.
- **Rust** `protocol/packet.rs` — `[u64; 32]` + `shift()` + 2 теста. **C#** `PacketCodec.cs`
  — `ulong[]` + `ShiftWindow`. **Kotlin** `PacketCodec.kt` — `LongArray` + `ushr`.

## H-5 — атомарная запись resolv.conf без symlink-follow ✅
`client/dns.rs::write_atomic` — предсказуемое имя + symlink-следующий `std::fs::write`
давали локальному атакующему подмену цели. Фикс: **рандомное имя + `O_EXCL` + `O_NOFOLLOW`**
+ ретрай на коллизию. Rust-клиент.

## H-3 — санитизация значений в nft-правилах kill-switch ✅
`client/killswitch.rs` — `tun_if` валидируется как Linux-ifname (`[A-Za-z0-9_-]{1,15}`),
`ips` реформатируются из распарсенного `IpAddr`. Defense-in-depth. Rust-клиент.

## H-1 — привязка data-ключей к статической личности сервера ✅ (Rust + C# + Kotlin)
**Валидное крипто-замечание:** сессионные ключи 0.7.0 деривятся из эфемерных DH
(`ee`, `mlkem`); static-ключ сервера в них НЕ подмешан → сбой эфемерального RNG
раскрывает туннель (слабее Noise IK). **Фикс:** подмешать `es = X25519(client_eph,
server_static)` в KDF — атакующему нужно сломать ОБА DH И добыть приватный static-ключ.

**Дизайн:** отдельный KDF-вариант со своей HKDF-солью (`…-v1-static-bound` /
`…-v2-hybrid-static-bound`) — нет «тихого» интеропа bound↔unbound: рассинхрон даёт
разные ключи и честный отказ. `es` симметричен (сервер: `static_priv × client_eph`;
клиент: `client_eph_priv × pinned_static`) → обе стороны выводят одинаковые ключи.

**Реализация (байт-в-байт):**
- **Rust** `crypto/derive.rs` (`derive_keys_bound`/`derive_keys_hybrid_bound` + e2e-тест
  `bound_handshake_keys_agree_end_to_end`), сервер `handler.rs`/`udp_handler.rs`, клиент
  `client/mod.rs` (`static_es` + все KDF-сайты), конфиг `config/{server,client}.rs`.
- **C#** `qeli-shared/Crypto/KeyDerivation.cs` (bound-варианты), `Model/VpnConfig.cs`
  (флаг + парсинг), `Vpn/VpnTunnelBase.cs` (`StaticEs` + 4 сайта).
- **Kotlin** `crypto/KeyDerivation.kt`, `model/Config.kt`, `QeliService.kt` (`staticEs` +
  `makeCodecs*` с `es`). Готча: `clientKeyPair.privateKey` — `java.security.PrivateKey`.

### ⚠️ С 0.7.1 H-1 ВКЛЮЧЁН ПО УМОЛЧАНИЮ
Флаг `auth.bind_static_to_session` (сервер) и `bind_static` (клиент, секция `[qeli]`)
**по умолчанию `true`**. Следствия:
- **WIRE-BREAKING**: сервер с H-1 принимает только клиентов на H-1 с запиненным ключом.
  Любой не-H-1 клиент (TOFU + весь задеплоенный 0.7.0-флот) перестаёт подключаться, пока
  не апгрейднется. **Апгрейдить сервер и всех клиентов синхронно.**
- **Беспиновый/TOFU** клиент (нулевой `key`) обязан явно поставить `bind_static = false`,
  иначе хендшейк падает с понятной ошибкой (нельзя вывести `es` без реального ключа).
  Шаблоны `client.conf` и `release/keenetic/client.conf.example` помечены `bind_static = false`.
- Дефолтные **secure-серверы** (`server-maxobf`, `server-reality`, `prod-*`) и пиновые
  клиенты несут флаг явно. Линки `qeli://` флаг не несут — клиент включает H-1 по своему
  дефолту (on), пин едет в линке.
- Для поэтапного апгрейда с legacy 0.7.0 ставьте `false` с обеих сторон, затем включайте.

## Ложные/незначимые пункты аудита (для справки)
- **C-1/C-2** (obfs) — XOR/ChaCha20-маскировка, это слой анти-DPI, а не слой
  безопасности (он поверх настоящего AEAD-туннеля); «слабая обфускация» — by design.
- **C-3** — `ReplayGuard` подключён (`server/reality.rs:84`).
- **M-9** — DNS-LB корректен. **M-14** — reset окна не даёт replay.
- **SHA-1** — это WebSocket-accept по RFC 6455, не крипто. **JOIN_MAGIC** — внутри AEAD,
  не на проводе. **thread_rng** — только для nonce-seed; ключи на `OsRng`.
- **M-4** (O(n) пул), **M-6** (IPv6 в split_host_port) — minor/by-design, по желанию.
