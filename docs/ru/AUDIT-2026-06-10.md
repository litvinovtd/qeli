## D1 — 🔴 Сервер не пишет в `/etc/qeli` под штатным systemd-юнитом

**Файлы:** `qeli/debian/qeli.service`, `qeli/src/server/mod.rs` (`generate_profile_key`),
`qeli/src/config/server.rs` (`default_users_file`), `qeli/src/server/control.rs`.

**Проблема.** `ProtectSystem=full` монтирует `/etc` только для чтения, но сервер по
умолчанию пишет туда: (1) генерация identity-ключа `/etc/qeli/identity/<name>.key`
при первом старте → `EROFS` → профиль не биндится; (2) `users.save()` из панели и
control-команд (`disable-user`/`enable-user`/`set-bandwidth`) → `EROFS`, причём
`control.rs` логировал ошибку, но возвращал `ok:true` («тихий успех»).

**Подход (внесено).** В юнит добавлен `ReadWritePaths=/etc/qeli` — пути ключей **не
менялись** (боевой identity-ключ не регенерируется, пиннинг клиентов цел), остальная
ФС остаётся read-only. `control.rs`: три команды теперь возвращают `ok:false` с
понятным текстом, если сохранение users-файла провалилось.

**Приёмка:** на чистой машине под юнитом `systemctl start qeli` поднимает профиль и
генерирует ключ; `qeli disable-user X` при недоступном для записи users-файле
возвращает ошибку, а не `OK`.

## W2 — 🟠 Argon2 веб-панели блокировал async-рантайм

**Файлы:** `qeli/src/web/auth.rs`, `qeli/src/web/api/login.rs`, `qeli/src/web/pages/*.rs`.

**Проблема.** `verify_credentials`/`is_authed`/`check_auth` исполняли Argon2
синхронно прямо в async-обработчике (на каждом `POST /api/login` и на каждом
API-запросе с HTTP Basic). Многопоточный tokio-рантайм: несколько одновременных
запросов забивают Argon2-работой все воркеры → стопорится супервизор (панель + реле
control-сокета). VPN-путь сделан правильно (`handler.rs` — `spawn_blocking`).

**Подход (внесено).** `is_authed`/`check_auth`/`verify_credentials` стали `async`;
Argon2 вынесен в `spawn_blocking`. Дешёвая cookie-проверка (HMAC) осталась inline.
Парсинг Basic вынесен в синхронный `basic_credentials` (без Argon2). Все вызовы
(`AuthGuard` + 5 страниц + login) переведены на `.await`.

**Приёмка:** всплеск запросов на `/api/login` не подвешивает панель; cookie-путь без
лишнего thread-hop.

## W1 — 🟠 Нет защиты от перебора на входе в панель

**Файлы:** `qeli/src/web/api/login.rs`, `qeli/src/web/mod.rs`.

**Проблема.** `login` не использовал `FailedAuthTracker` (в отличие от VPN-auth).
CSRF-middleware не мешает целенаправленному перебору через curl.

**Подход (внесено).** Логин переиспользует существующий `failed_auth` (в
`ServerState` супервизора), ключ — username + source-IP через `ConnectInfo`. При
лимите — `429`. Локти трекера не удерживаются на время Argon2. В `web/mod.rs`
включён `into_make_service_with_connect_info::<SocketAddr>()`.

**Приёмка:** N неудачных логинов (дефолт `brute_force`: 5/15 мин) → `429`.
**Прим.:** покрыт только `/api/login`; Basic-auth путь — см. **W1b**.

## U1 — ❎ UDP-усиление/отражение — ЛОЖНАЯ НАХОДКА (уже реализовано)

**Файлы:** `qeli/src/server/udp_handler.rs` (`handle_new_udp_client`),
`qeli/src/client/mod.rs`, `qeli-shared/.../VpnTunnelBase.cs`, `qeli-android/.../QeliService.kt`.

При первичном аудите я **не дочитал** `handle_new_udp_client` и ошибочно записал это в
находки. На деле anti-amplification **уже реализован**: сервер отвергает первый UDP-
датаграмм `< MIN_UDP_INITIAL (1200B)` с явным «anti-amplification guard»
(`handle_new_udp_client:692-703`), а все клиенты паддят UDP-ClientHello до ≥1200B —
Rust (`client/mod.rs:1524-1527`, `build_client_hello(..., 1200, ..)`), C#
(`EstablishAndRun(..., padToMin: 1200, isUdp: true)`), Kotlin
(`establishAndRun(.., padToMin = 1200, isUdp = true)`). TCP-путь не паддит (там
amplification неприменим). Правило 3× (3×1200 ≥ ответ рукопожатия) соблюдено.
**Действий не требуется.**

## U2 — 🟡 UDP-флуд состояния подменой source-IP (✅)

**Файлы:** `qeli/src/server/udp_handler.rs`.

Рейт-лимит только для новых сессий и по `addr.ip()`; подменяя случайные IP, можно
плодить `AwaitingAuth`-записи (чистятся по `handshake_timeout`). **Внесено:** константа
`MAX_PENDING_HANDSHAKES = 1024` (на воркер); при достижении cap перед вставкой новой
half-open сессии вытесняется самая старая `AwaitingAuth` (аутентифицированные не
затрагиваются). **Не закрывает U1** (усиление) — только память.

## C1 — 🟡 Win: учётные данные сервиса восстановимы локально (✅)

**Файлы:** `qeli-win/QeliWin/Service/ServiceState.cs` (`SaveProfile`/`RestrictProfileAcl`).

Профиль (пароль+obfs_key) шифруется DPAPI `LocalMachine` (расшифрует любой локальный
процесс) в `%ProgramData%\QeliWin`, каталог наследует дефолтный ACL (у `Users`
чтение). **Внесено:** ACL ставится на сам **файл** профиля в `SaveProfile` (а не на
каталог — иначе он осциллировал бы между GUI-юзером и SYSTEM-сервисом): DACL =
`текущий-юзер (writer) + SYSTEM (reader) + Administrators`, наследование снято. Всё в
`try/catch` — сбой ACL не ломает сохранение (DPAPI-шифрование остаётся). Сборка
подтвердила: `FileInfo.SetAccessControl`/`FileSecurity`/`WellKnownSidType` резолвятся на
net10-windows. **Остаётся ручной e2e на Windows:** запись профиля из GUI (юзер) +
чтение сервисом (SYSTEM) под новым ACL.

## W1b — 🟡 Рейт-лимит не покрывает Basic-auth (✅)

**Файлы:** `qeli/src/web/auth.rs` (`AuthGuard`).

W1 покрывает форму логина; брутфорс через HTTP Basic на API-эндпоинтах без локаута
(W2 убрал DoS-усиление, но не перебор). **Внесено:** `AuthGuard` теперь — cookie
(дешёво) → иначе Basic с рейт-лимитом через `failed_auth`+`ConnectInfo`. Неудача
считается **только** когда Basic-креды предъявлены и неверны; запрос без заголовка
Authorization и невалидная cookie НЕ считаются (иначе любой залочил бы админа). Локти
не удерживаются на время Argon2.

## L1–L3, T1 — ⚪ мелочи

- **L1** (✅¹) lockout-DoS: атакующий с любого IP блокирует известный username (`mod.rs`
  `FailedAuthTracker`). На 2026-06-10 оставлено как policy-решение; **¹закрыто 2026-06-11**
  — жёсткий username-лок заменён на adaptive tarpit (`check_ip` + `user_tarpit`),
  см. [`AUDIT-2026-06-11.md`](AUDIT-2026-06-11.md) § L1.
- **L2** (✅) гонка прав control-сокета: родительский каталог теперь
  выставляется в `0700` ДО `bind` (`control.rs`), так что во время окна между bind и
  chmod 0600 сокет недостижим (каталог гейтит обход).
- **L3** (✅) C#/Kotlin-декодеры (`qeli-shared/.../PacketCodec.cs`,
  `qeli-android/.../PacketCodec.kt`): добавлена явная проверка
  `payloadLen ≥ Nonce+Tag+Counter+2` и полноты пакета (как в Rust) → чистое
  `PacketException`, а не сырое index-исключение на битом/усечённом рекорде.
- **T1** (✅) `scripts/rebuild_apk.py` хардкодил `/root/gradle-8.11.1`, но AGP 9
  требует Gradle ≥9.4.1 → сборка APK падала на apply-плагине. Переключено на project
  Gradle wrapper (9.5.1, уже в кэше .11). Проверено: `assembleDebug` BUILD SUCCESSFUL.
