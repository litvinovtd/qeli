# qeli — разбор второго внешнего аудита (2026-06-11)

Источник: второй внешний аудит qeli v0.6.0 (принесён пользователем, отдельно от
[`AUDIT-2026-06-11.md`](AUDIT-2026-06-11.md)). Каждый пункт сверен с кодом.

> **Мета-вывод: аудит в основном галлюцинирован.** Многие утверждения опровергаются
> прямой проверкой репозитория — уверенно описывает несуществующие «заглушки» и
> отсутствующие CI-джобы. Когда такие базовые факты неверны, доверять остальному без
> проверки нельзя. Из ~30 пунктов **проверяемо ложных ~13** (включая оба «🔴»),
> большинство остального — by-design или тривиальности. Реально обоснован практически
> один пункт (`panic=abort`), и он лишь усиливает приоритет уже известного T2.

> **Верификация фиксов (2026-06-11):** внесены 2 гигиена-правки (см. ниже); прогнаны в
> общем лаб-гейте — Rust (.10) `build`/`test` **188 passed**/`clippy -D warnings`/`fmt`
> — всё зелёное.

## Полный разбор

### ❌ Ложные (с доказательством)

| Claim | Факт |
|---|---|
| `thread_rng()` «не крипто-безопасный» | `rand::thread_rng()` — это **CSPRNG** (ChaCha, сид из OS). Фактическая ошибка о крейте |
| rate-limiter «блокирующий std Mutex в async через await» (mod.rs:1307) | Это `tokio::sync::Mutex` (`.lock().await`), держится микросекунды, **не** через await |
| exchange.rs «не проверяет 5 low-order точек» | Проверка **all-zero результата** = способ RFC 7748 §6.1 отсечь все low-order точки; отдельный блок-лист не нужен. Документировано + тест `derive_shared_checked_rejects_low_order_point` |
| cookie «нет Secure/HttpOnly/SameSite» | `login.rs` — `HttpOnly; SameSite=Strict` + условный `Secure` (форсить Secure на HTTP сломало бы cookie) |
| `build_auth_ok` «утечка, OK: без JSON» (🔴) | `serde_json::to_string` на `json!`-`Value` практически не фейлится — ветка недостижима (= T5 из первого аудита) |
| REPLAY_WINDOW=64 «ложные срабатывания на 100k пакетов» | Непонимание: окно **скользит** с каждым пакетом — это допуск на reordering в 64 пакета, не лимит на总число |
| UDP-writer «висячий на мёртвом канале» (🔴) | `writer_rx.recv()` на закрытом канале → `None => break`; зависнуть нельзя |
| recv_peek «не cancellation-safe» | `stream.peek().await` cancel-safe; отмена таска дропает future и прерывает peek немедленно |
| control-socket «два воркера конкурируют, нужен SO_REUSEADDR» | Сокет биндит **один** супервизор; для Unix-сокета SO_REUSEADDR так не работает; окно прав закрыто 0700-локом каталога (L2) |

### ⚪ By design / не дефект

- **fake-tls cert без chain-verify** — *сами пишут «by design»*. Безопасность fake-tls даёт
  не сертификат, а X25519-proof + channel-binding; для настоящего TLS есть `reality-tls`.
- **v1→v2 proof «нет миграции»** — v2 это намеренный security-апгрейд (channel-binding),
  единая версия 0.6.0 на всех компонентах, v1-клиентов не существует.
- **TUN strip → silent drop** — пропуск **метрики**, не баг; и только в TAP (не подключён).
- **hot-reload конфига** — частично неверно: users/brute-force **горячо** перезагружаются
  по SIGHUP (`WorkerCmd::ReloadUsers`); рестарт нужен лишь для listener/crypto-изменений.
- **Нет prometheus/structured-логов** — честная roadmap-дыра, не дефект.

### 🟡 Реально стоит внимания — и внесено

- **`panic = "abort"` (Cargo.toml)** — единственный по-настоящему полезный пункт. Скорее
  всего **намеренный** (раскрутка паники через FFI/JNI-границу — UB, abort безопаснее).
  Следствие верное: паника в таске убивает весь worker (супервизор перезапустит, но
  активные сессии отвалятся) → усиливает приоритет **T2** (триаж достижимых `unwrap/expect`).
  **Триаж проведён:** на сетевом пути **достижимых паник НЕТ** — все non-test `unwrap/expect`
  инфолибильны на фикс-размерах (crypto-ключи, HKDF/HMAC) / startup-конфиг (fail-fast) /
  send-path. Путь разбора входящих байтов panic-free by construction. Менять `panic=abort`
  не требуется.

#### Внесённые гигиена-фиксы

**1. Портируемость `set_tcp_keepalive`** (`qeli/src/transport/tcp.rs`). Linux-специфичные
`TCP_KEEPIDLE`/`TCP_KEEPINTVL`/`TCP_KEEPCNT` теперь под `#[cfg(target_os = "linux")]` с
no-op фолбэком для прочих таргетов; `use AsRawFd` перенесён внутрь Linux-ветки (на верхнем
уровне ломал бы non-Linux компиляцию). Практического бага не было (крейт собирается только
под Linux/musl, десктоп-клиенты — C#), это гигиена.

**2. Единообразие poisoned-lock** (`qeli/src/server/reality.rs`). `reality_borrow` читается
через `read().unwrap_or_else(|e| e.into_inner())` (recover-from-poison, как `lock_or_recover`
/T6), а не `expect`. Под `panic=abort` ветка moot (паника-под-локом аборт­ит процесс раньше
отравления), но паттерн теперь единый и корректен при смене panic-стратегии.

### ⬜ Тюнинг-заметка (не внесено)

- **REPLAY_WINDOW=64 для UDP** — обоснование аудита неверно, но 64-пакетное окно reordering
  невелико для высокобитрейтного UDP (WireGuard держит ~8192-битное). Если появятся
  спорадические дропы на UDP-линках — расширить. Не баг; трогать рабочую anti-replay крипто
  без доказанной нужды не стоит.
